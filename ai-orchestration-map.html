AI Engineering Orchestration Map

How `.ai` and `AGENTS.md` organize repo groups, repos, expert roles, SDLC gates, logs, and impact reports.

Hierarchy Startup Demo SDLC Roles Reports

Repo Group -> Repo -> Role

Repo groups coordinate many company/domain repos. Repos run SDLC phases. Roles execute expert work under the same global rules.

Layer 1

Repo Group

Workspace for many companies, repos, products, services, or domains.

  • Owns shared rules
  • Tracks cross-repo impact
  • Escalates unresolved conflicts
Layer 2

Repo

Company or domain unit that runs the full software development cycle.

  • Owns phase logs
  • Owns repo decisions
  • May declare child repos
Layer 3

Role

Domain expert agent with character, skills, responsibilities, hooks, and rules.

  • Owns specialist output
  • Creates handoff notes
  • Reports impact outside scope

Worker Startup

Markdown gives the worker behavior. YAML gives the worker configuration. A new project edits `.ai/project.yml` first.

AGENTS.md Root entrypoint. Tells every AI worker that `.ai` controls the workflow.
manifest.yml Declares hierarchy, load order, runtime folders, and hard requirements.
SKILLS.md Human-readable worker behavior: logging, gates, handoffs, impact reports.
project.yml Project identity: repo group, repo, current phase, active roles, environments.
role/*.yml Specialist instructions for orchestrator, product, design, engineering, QA, and ops.

Logs and Impact Reports

The goal is not just to finish work. The goal is to finish each phase with evidence and report anything that affects another role or repo.

Phase starts Create phase log from `.ai/templates/phase-log.md`.
Role works Use role character, skills, responsibilities, hooks, and rules.
Impact check If another role or repo is affected, create impact report.
Gate review Check DoD item by item. No bulk approval.
Handoff Write handoff note and link evidence before next phase.

Non-Negotiables

These rules apply across every repo group, repo, role, phase, and emergency path.

Delivery

  • Every phase must end with a phase log.
  • DoD is checked item by item.
  • DEV -> STAGING -> UAT -> PROD is the default flow.
  • CI, peer review, and secret scanning are required.

Coordination

  • Cross-role impact requires a report.
  • Cross-repo impact requires a group report.
  • Orchestrator resolves unresolved conflicts.
  • P0 and P1 incidents require post-mortems within 24 hours.
