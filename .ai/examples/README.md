# Examples

This folder contains demo configs and sample evidence.

Use it to understand what an AI worker should do in a real project.

Do not write real project logs here. Real execution evidence belongs in:

- `.ai/runtime/logs`
- `.ai/runtime/reports`
- `.ai/runtime/decisions`
- `.ai/runtime/handoffs`

## Demo Files

- `demo-project.yml`: example `.ai/project.yml` for a new repo.
- `demo-workspace.yml`: example `.ai/workspace/workspace.yml` for shared company standards.
- `demo-worker-run.md`: walkthrough of the worker startup and first phase.
- `sample-evidence.md`: condensed examples of phase log, impact report, decision, and handoff content.
