# UX Designer Checklist

## Before Start

- [ ] OKR or PRD
- [ ] Target users
- [ ] Acceptance criteria
- [ ] Platform constraints
- [ ] Known accessibility requirements

## During Work

- [ ] Current phase log is created or updated.
- [ ] Work is traceable to OKR, AC, ticket, incident, or user request.
- [ ] Decisions are recorded when they affect future work.
- [ ] Tests, review, metrics, security, or compliance evidence is captured where relevant.

## Must Produce

- [ ] Research synthesis
- [ ] User flow
- [ ] Wireframes or high fidelity design
- [ ] State matrix
- [ ] Accessibility notes
- [ ] Design handoff

## Impact Check

- [ ] Checked: Design changes acceptance criteria
- [ ] Checked: Design requires new API or analytics events
- [ ] Checked: Accessibility risk is discovered
- [ ] Checked: Compliance-sensitive data appears in the flow

If any checked item is true, create an impact report before handoff.

## Must Not Do

- [ ] Did not: Hand off screens with missing loading, empty, error, success, or permission states
- [ ] Did not: Ignore WCAG expectations
- [ ] Did not: Change product scope silently

## Handoff

- [ ] Handoff note created.
- [ ] Next owner is named.
- [ ] Done and not done are explicit.
- [ ] Risks and blockers are explicit.
- [ ] Required next action is clear.
