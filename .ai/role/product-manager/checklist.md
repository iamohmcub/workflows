# Product Manager Checklist

## Before Start

- [ ] Business context or user request
- [ ] Target user or customer segment
- [ ] Known constraints and stakeholder commitments
- [ ] Metric baseline or Data Analyst owner identified

## During Work

- [ ] Current phase log is created or updated.
- [ ] Work is traceable to OKR, AC, ticket, incident, or user request.
- [ ] Decisions are recorded when they affect future work.
- [ ] Tests, review, metrics, security, or compliance evidence is captured where relevant.

## Must Produce

- [ ] Outcome-focused objective
- [ ] Measurable key results
- [ ] PRD or scope note
- [ ] Acceptance criteria
- [ ] Product risks and assumptions

## Impact Check

- [ ] Checked: Scope changes after downstream work starts
- [ ] Checked: Success metric changes
- [ ] Checked: Customer promise or SLA changes
- [ ] Checked: Another repo is needed for product delivery

If any checked item is true, create an impact report before handoff.

## Must Not Do

- [ ] Did not: Ask engineers to implement before acceptance criteria exist
- [ ] Did not: Change scope mid-phase without impact report
- [ ] Did not: Define vague success metrics that cannot be measured

## Handoff

- [ ] Handoff note created.
- [ ] Next owner is named.
- [ ] Done and not done are explicit.
- [ ] Risks and blockers are explicit.
- [ ] Required next action is clear.
