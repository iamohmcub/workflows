# Legal Compliance Checklist

## Before Start

- [ ] Data flow
- [ ] Jurisdictions or customer commitments
- [ ] Vendor list
- [ ] Retention expectations
- [ ] User-facing copy or policy impact

## During Work

- [ ] Current phase log is created or updated.
- [ ] Work is traceable to OKR, AC, ticket, incident, or user request.
- [ ] Decisions are recorded when they affect future work.
- [ ] Tests, review, metrics, security, or compliance evidence is captured where relevant.

## Must Produce

- [ ] Compliance requirements
- [ ] Risk assessment
- [ ] Retention notes
- [ ] Vendor or license decision
- [ ] Approval or block note

## Impact Check

- [ ] Checked: Personal data handling changes
- [ ] Checked: New vendor is introduced
- [ ] Checked: Regulated workflow changes
- [ ] Checked: License risk appears
- [ ] Checked: Accessibility compliance risk is found

If any checked item is true, create an impact report before handoff.

## Must Not Do

- [ ] Did not: Leave legal guidance as vague policy language
- [ ] Did not: Approve unclear PII handling
- [ ] Did not: Treat compliance as a post-launch checklist

## Handoff

- [ ] Handoff note created.
- [ ] Next owner is named.
- [ ] Done and not done are explicit.
- [ ] Risks and blockers are explicit.
- [ ] Required next action is clear.
