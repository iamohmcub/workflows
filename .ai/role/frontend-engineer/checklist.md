# Frontend Engineer Checklist

## Before Start

- [ ] UX spec and state matrix
- [ ] Acceptance criteria
- [ ] API contract
- [ ] Copy or localization guidance
- [ ] Analytics event requirements

## During Work

- [ ] Current phase log is created or updated.
- [ ] Work is traceable to OKR, AC, ticket, incident, or user request.
- [ ] Decisions are recorded when they affect future work.
- [ ] Tests, review, metrics, security, or compliance evidence is captured where relevant.

## Must Produce

- [ ] Component implementation
- [ ] State coverage
- [ ] Frontend tests
- [ ] Accessibility evidence
- [ ] Performance notes
- [ ] UI risk notes

## Impact Check

- [ ] Checked: API contract does not match UI need
- [ ] Checked: Design state is missing
- [ ] Checked: Analytics event changes
- [ ] Checked: Accessibility blocker is found
- [ ] Checked: Shared component affects another repo

If any checked item is true, create an impact report before handoff.

## Must Not Do

- [ ] Did not: Ignore required UI states
- [ ] Did not: Hard-code user-facing strings when copy ownership or localization exists
- [ ] Did not: Change API contract without Tech Lead approval and impact report

## Handoff

- [ ] Handoff note created.
- [ ] Next owner is named.
- [ ] Done and not done are explicit.
- [ ] Risks and blockers are explicit.
- [ ] Required next action is clear.
