# Product Owner Checklist

## Before Start

- [ ] PRD or approved scope
- [ ] Acceptance criteria
- [ ] Priority order
- [ ] Dependencies and constraints

## During Work

- [ ] Current phase log is created or updated.
- [ ] Work is traceable to OKR, AC, ticket, incident, or user request.
- [ ] Decisions are recorded when they affect future work.
- [ ] Tests, review, metrics, security, or compliance evidence is captured where relevant.

## Must Produce

- [ ] Ready backlog items
- [ ] Acceptance notes
- [ ] Clarification log
- [ ] Rejected work notes
- [ ] Sprint readiness report

## Impact Check

- [ ] Checked: Acceptance criteria are ambiguous
- [ ] Checked: Reprioritization affects committed work
- [ ] Checked: Story split changes scope
- [ ] Checked: A downstream role is blocked by product uncertainty

If any checked item is true, create an impact report before handoff.

## Must Not Do

- [ ] Did not: Accept incomplete work
- [ ] Did not: Change product strategy without Product Manager approval
- [ ] Did not: Bypass acceptance criteria to unblock a sprint

## Handoff

- [ ] Handoff note created.
- [ ] Next owner is named.
- [ ] Done and not done are explicit.
- [ ] Risks and blockers are explicit.
- [ ] Required next action is clear.
