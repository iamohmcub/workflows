# Data Analyst Checklist

## Before Start

- [ ] OKR or key result
- [ ] User segment
- [ ] Event source or data model
- [ ] Baseline availability
- [ ] Decision question

## During Work

- [ ] Current phase log is created or updated.
- [ ] Work is traceable to OKR, AC, ticket, incident, or user request.
- [ ] Decisions are recorded when they affect future work.
- [ ] Tests, review, metrics, security, or compliance evidence is captured where relevant.

## Must Produce

- [ ] Metric definition
- [ ] Instrumentation requirements
- [ ] Baseline report
- [ ] Dashboard or query plan
- [ ] Analysis notes
- [ ] Data quality flags

## Impact Check

- [ ] Checked: Event schema changes
- [ ] Checked: Data quality blocks measurement
- [ ] Checked: Metric baseline is unavailable
- [ ] Checked: Analytics changes affect another repo
- [ ] Checked: Experiment validity is at risk

If any checked item is true, create an impact report before handoff.

## Must Not Do

- [ ] Did not: Report correlation as causation
- [ ] Did not: Accept low-quality data silently
- [ ] Did not: Allow launch success to remain unmeasured

## Handoff

- [ ] Handoff note created.
- [ ] Next owner is named.
- [ ] Done and not done are explicit.
- [ ] Risks and blockers are explicit.
- [ ] Required next action is clear.
